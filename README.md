# 工藝資源地圖

社群共筆的工藝資源通訊錄。陶瓷先上線，未來可擴充金工、木工等領域。
任何人都能在網站上投稿新增、或協助維護（回報有誤、補充建議），但看不到也改不了
別人的原始資料；維護者只要把資料的「狀態」設為 `approved` 即可公開。

## 目錄結構

```
craft-map/
├── index.html              ← 網頁主檔（HTML 結構）
├── assets/
│   ├── css/
│   │   └── style.css       ← 樣式
│   └── js/
│       ├── config.js       ← 設定（平常只改這裡：試算表 CSV、後端網址）
│       ├── data.js         ← 內建備援資料（自動轉出，平時免動）
│       └── app.js          ← 程式邏輯（篩選、搜尋、渲染、站內表單，一般免動）
└── README.md               ← 本說明
```

## 快速開始

直接用瀏覽器打開 `index.html`。預設會讀線上試算表；若想單純看內建資料，
把 `config.js` 的 `SHEET_CSV_URL` 暫時清空即可改用 `data.js`。

## 資料欄位（試算表第一列標題）

程式是「依標題對應」讀寫，欄位順序可調、也能再加新欄位。目前欄位：

- `領域`：如 陶瓷、金工、木工。
- `分類`：可複合——一家跨多類用**全形 `｜`** 隔開，例如 `設備｜原料、土、陶藝工具`；
  前端會自動讓它出現在每個類別的篩選下。（類別名稱本身用「、」，所以分隔符不能用「、」。）
- `名稱`、`地址`、`聯絡方式`、`營業項目`：基本資訊。
- `建議`：卡片底部的小提醒（如「建議先電話預約」）；多項用「、」串接。
- `狀態`：空白＝待審核（不顯示）；填 `approved` 才公開。

縣市不需獨立欄位，會由地址自動推算。

## 設定（config.js）

```js
window.CONFIG = {
  SHEET_CSV_URL: "https://docs.google.com/.../pub?output=csv", // 試算表發布的 CSV；留空改用 data.js
  SUBMIT_URL:    "https://script.google.com/macros/s/.../exec", // Apps Script 後端（站內新增／維護）
  UPCOMING_DOMAINS: ["金工", "木工"],   // 顯示為「即將推出」的領域頁籤
  FORM_URL: "",          // 備援：沒設 SUBMIT_URL 時，「新增」改開的 Google 表單
  REPORT_FORM_URL: ""    // 備援：沒設 SUBMIT_URL 時，「維護」改開的 Google 表單
};
```

詳細的試算表、表單與後端建立步驟，見上層資料夾的
「串接Google試算表_設定指南.md」與「站內新增_設定指南.md」。

## 功能

- **站內新增**：右下角「＋ 我發現一家」。分類可複選、找不到可在「其他」自填；
  建議用分組下拉（聯絡方式／付款方式／訂購方式／預約營業）各選一項，也可自填。
- **協助維護**：每張卡片的「協助維護」。可回報有誤／搬遷，或補充建議；
  全部選填，至少填一項即可。回報進「回報審查」分頁，由維護者確認。
- **資料驅動**：領域頁籤、分類篩選都依資料自動生成，不寫死，方便擴充。

## 部署

純前端靜態網站，可放上 GitHub Pages / Netlify / Cloudflare Pages。
GitHub Pages：把整個 `craft-map` 內容推到 repo，Settings → Pages → 由 main 分支 / root 發布。

需要注意：

- 改了 Apps Script（`.gs`）後，要在編輯器「部署 → 管理部署作業 → 編輯 → 版本選**新版本** → 部署」
  才會生效；只按儲存的話 `/exec` 仍跑舊版。
- 試算表的「資料」分頁與「回報審查」分頁都要有一欄標題正好是 **`建議`**，建議才會被存進去。

## 設計原則

- **只增不刪改**：投稿與維護都只會新增資料，看不到也改不了別人的原始資料。
- **先審後上**：狀態空白時不顯示，維護者填 `approved` 才公開。
- **可擴充領域**：領域是資料欄位而非寫死分頁；新增金工／木工只要在資料加值、
  並在 `index.html` 把對應頁籤的 `soon` 拿掉即可。
